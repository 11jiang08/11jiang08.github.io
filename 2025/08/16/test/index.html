线段树是用于在区间上进行信息统计的二叉树。其思想可以概括为 “分治法+二叉树+Lazy-Tag”。

线段树节点所表示的 “线段” 的值，可以是区间和，最值或是其它根据题目灵活定义的值。线段树高效的原因在于：每个节点的值，都代表以它为根的子树上所有节点的值，那么，在查询这棵子树所代表的区间的值时，就不必遍历整棵子树，而是直接读取这颗子树的根植。

线段树合适解决的问题的特征是：大区间的解可以由小区间的解合并而来。

线段树的性质

  1. 每个节点都代表一个区间。
  2. 有唯一的根节点，代表整体区间
  3. 每个夜间点代表长度为 \(1\) 的单位区间
  4. 出叶节点和根节点之外的内部节点 \([l,r]\)，取 \(mid=\lfloor\frac{1+r}{2}\rfloor\)，左子节点为 \([l,mid]\)，右子节点为 \([mid+1,r]\)。
  5. 除最后一层，线段树为完全二叉树，深度为 \(\log n\)。
  6. 如果用数组来保存线段树，长度需要 \(4n\)，故会有空间上的浪费。

建树

对于区间 \([p,l,r]\)，\(p\) 表示节点编号，\(l\) 为区间左端点，\(r\) 为区间右端点。

如果 \(l=r\)，说明已经递归到了叶子节点，直接把对应的区间值放进去即可，然后返回。

否则，将区间分成左右两部分进行递归。取 \(mid=\lfloor\frac{1+r}{2}\rfloor\)，左子节点为 \([l,mid]\)，右子节点为 \([mid+1,r]\)。在左右子节点都完成递归之后，按照题意对该节点的数值进行维护，然后返回。

单点修改

线段树的单点修改遵循以下步骤：

假设修改第 \(x\) 个节点，从根节点出发，递归找到区间 \([x,x]\) 的节点，更新值。

从下往上，依次更新父节点的信息，维护线段树的性质。

区间查询

对于一个区间 \([L,R]\) 的查询，必定能分成不超过 \(\log n\) 个子区间进行分别查询。

如果现在查到了区间 \([l,r]\)，分为以下几种情况：

  1. 如果 \(L \leq l \leq r \leq R\)，则直接返回该区间的值。
  2. 如果 \(r<L\) 或 \(l>R\)，说明两个区间并无交集，直接退出。
  3. 否则将区间 \([l,r]\) 分为区间 \([l,mid]\) 和 \([mid+1,r]\)，分别查询。

时间复杂度为 \(O(\log n)\)。

如果想要优化，则在递归前判断 \([l,mid]\) 和 \([mid+1,r]\) 与区间 \([L,R]\) 是否有交集，这样不用处理情况 \(2\)。

区间修改

在区间修改时，如果某个节点 \(P\)，被修改区间 \([l,r]\) 全部覆盖，那么 \(P\) 和 \(P\) 的子树全部要修改，时间复杂度最高为 \(O(n)\)。

而且，如果 \(P\) 修改了之后没有被查询到，那么修改 \(P\) 就是徒劳的。

lazy tag

很容易想到解决办法，还是利用线段树的特征：lazy tag 给 \(P\) 节点增加标记，代表 “此节点有更新，但子节点尚未被更新”，依次统一记录区间 \(P\) 的修改，而不是依次修改区间内的每个元素。

如果在后续指令中，需要从节点 \(P\) 向下递归，那么：

  1. 检查标记状态，根据标记更新 \(P\) 的子节点。
  2. 给 \(P\) 的子节点更新标记状态。
  3. 删除 \(P\) 的标记状态。

这样做，区间修改的复杂度就降至 \(O(\log n)\)。

在 \(modify\) 和 \(query\) 时都需要 \(pushdown\)。

区间最值

修改区间最值指的是这样一类操作：对于一个序列 \(a\)，给出 \(L,R,x\)，将 \([L,R]\) 范围内的每一个 \(a_i\) 修改为 \(a_i=\max(a_i,x)\) 或 \(a_i=\min(a_i,x)\)。

对于线段树的每个节点，定义 \(4\) 个标记：区间和 \(sum\)，区间最大值 \(ma\)，区间严格次大值 \(se=-1\)，区间最大值个数 \(t\)。

此处以将区间 \([L,R]\) 中的每一个值修改为 \(a_i=\min(a_i,x)\) 为例：

根据 \([L,R]\) 定位到线段树的节点，搜到某个节点时，会出现以下三种情况：

  1. \(ma \le x\)：此次修改不影响节点，退出。
  2. \(se<x<ma\)：此次修改影响最大值，更新 \(sum=sum-t·(ma-x)\)，\(ma=x\)，然后退出。
  3. \(se \le x\)：此时无法直接更新这个节点，递归它的左右儿子。

上述算法的关键是严格次大值 \(se\)，它起到了剪枝的作用。

每次修改的时间复杂度为 \(O(\log n)\)。

区间历史最值

P6242 是一道区间历史最值的模板题。

题目简介

给你一个长度为 \(n\) 的序列 \(a\)。最开始令序列 \(b=a\)。

现在进行 \(m\) 次操作，分为 \(5\) 种：

  • 1 l r k：将 \(a_l \sim a_r\) 加上 \(k\)。
  • 2 l r v：将 \(a_l \sim a_r\) 中的每个数对 \(v\) 取最小值。
  • 3 l r：求 \(\displaystyle \sum_{i=l}^r a_i\)。
  • 4 l r：求 \(\displaystyle \max_{l \le i \le r}\{a_i\}\)。
  • 5 l r：求 \(\displaystyle \max_{l \le i \le r}\{b_i\}\)。

每次操作后令 \(b_i=\max(a_i,b_i)\) 。

思路

操作 \(1\) 和操作 \(3\) 非常模板，先考虑解决操作 \(4\)，求区间最大值。

每个节点维护一个 \(ma\) 表示该节点表示的区间的最大值，进行操作 \(1\) 时顺便把 \(ma\) 加上 \(k\)，标记下传时同理。

现在我们解决了操作 \(1,3,4\)。

接下来我们尝试解决操作 \(5\)，显然在这题中，\(b_i\) 就是 \(a_i\) 的历史最大值，操作 \(2\) 就是区间最小值操作，操作 \(5\) 是求区间最大历史最大值。

操作 \(5\) 的瓶颈在于区间加后无法快速更新序列 \(b\)。

因为一个区间总能被若干个点表示，我们先将一个区间先简化一个节点。

我们在区间加时，会对一个区间加上 \(k\)。

  • \(k>0\)，历史最大值可能被更新。

  • 否则，区间最大值不可能更新。

想象一下我们正在对一个节点（不是任意区间）进行疯狂的区间加，此时也就是该节点的 tag 在一直被改变。

经过一次加法，这个 tag 可能变大（\(k>0\)），也可能变小（\(k<0\)）。

但是我们注意到，那么在标记下传前，当前节点的儿子的值是不会变的。

那么显然，当tag 达到最大时，下传的时候才可以使儿子达到当前最大。而只有当前最大才有可能更新历史最大。

假设这个tag 为 tag1。

那么，我们只需要再设一个 tag2 来表示没下传前的最大 tag1，那么下传更新序列 \(b\) 就容易了。

注意下传时的顺序，要先更新 maxb 再更新 maxa，先更新 tag2 再更新 tag1。

可以过测试点 \(5\) 和 \(6\) 的代码，再往下就是完整的代码。

这里一定要理解透，不然后面肯定都看不懂。

对于操作 \(2\) 的解决见 区间最值。

代码及其理解

首先说明，本题解主要以代码为主。个人认为代码写得很清晰，如果懂得上述思路就一定可以看懂。

先来看一下线段树每个节点需要存些什么。

以下是需要维护的 tag：

  • add1：该区间最大值的懒标记。
  • add2：该区间非最大值的懒标记。
  • add3：该区间最大值的懒标记的最大值。
  • add4：该区间非最大的值的懒标记的最大值。

具体地，add3 是在未下传懒标记前最大的 add1 的值，add4 是在未下传懒标记前最大的 add2 的值。

这 \(4\) 个 tag 只要理解透了，代码就一定可以看得懂。

注意事项&常数优化

  1. 使用快读和快写。
  2. 求和开 long long。
  3. 只有求和才开 long long，其他用 int。这样可以比全用 long long 快半秒左右。
  4. 递归的时候参数尽量少。
  5. 用一个结构体来储存树的各种信息，这样空间访问较连续，速度较快。
  6. 在建树的时候读入，可以一定的节省空间和时间。
  7. 加一些函数前加 inline。
  8. 修改时的更新可以用 change 代替，可以减少大量码量。
  9. 注意更新顺序。

区间合并

线段树非常适合做区间合并。

线段树的兄弟节点之间有相邻关系，这个特性方便线段树做区间合并操作。
